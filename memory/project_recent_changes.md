Session of 2026-04-16 landed these in a single pass:

1. **Game renamed** to "SpaceShip" (was "SAD ABDUCTION"). Changed: `<title>` (line 8) and main-menu text fallback (~line 8540). Logo image `logo.png` is still the primary branding when loaded.

2. **All planets + asteroid unlocked by default.** `unlockedPlanets` now defaults to `planetDefs.map(p=>p.id)` in both `initPlanetProgress` and the load-game fallback. leaderRelations initialized for all 7 canon planets.

3. **New planet: Sun ("Helion")** — `isSun:true`, added to planetDefs, position `(60000, -48000)` in space (moved far-far-away 2026-04-18). Hidden until `p.discovered=true`, which is set when the ship enters `radius + 2500` of the sun's space position. Ship bounds expanded to `spaceWidth+60000` and `-spaceHeight-50000` so the player can actually reach it. Hidden from main space render + minimap until discovered. Sun is also selectable in the Debug → World planet grid for direct unit preview (it has alienTypes: ember, flare, inferno).

4. **Earth expanded to 34000 wide** with new biomes: farmland (6000-9500) with cows/sheep/farmers/barns/silos/farmhouses/haystacks, and snow (16000-19500) with wolves/yetis/snow-capped pines. Jungle now has tigers + parrots in addition to monkeys. Farmland has 2 green "Tractor" vehicles (rendered as green trucks). See project_earth.md for full biome table.

5. **Ocean: visible sandy seabed + much more life.** Seabed gradient brightened (#c8a868→#5a4020) with a surface highlight band and deterministic speckles. New creature types with full render branches: kelp (tall swaying), starfish, crab (walks on seabed, clamps dir at oceanBounds), sea turtle (swims), shark (mid-deep, menacing), seaRock (decorative), shell. Counts boosted: seaweed 25→60, coral 18→40, fishSchool 12→22, jellyfish 8→18, whales 1→2, anglerfish 2→4, shipwreck 1→2, treasure 3→5. Ocean bounds now derived from `earthBiomes.find('ocean')` into global `oceanBounds` which wrap logic uses (fishSchool, whale, anglerfish, crab, turtle, shark).

7. **Settings → rebindable controls with context tabs (2026-04-18).** Main-menu "Settings" opens a key-rebinding list. Each `KEY_ACTIONS` entry has a `context` of `'ship'`, `'foot'`, or `'both'`. The menu has three tabs (ALL / SHIP / ON-FOOT, switched with A/D) that filter the action list. Rebinds persist via `localStorage('sadabduction_keybinds')`; a `_physToCanon` map translates physical→canonical keys in the global keydown/keyup hook so game code still reads canonical keys (e.g. `keys['a']`). "PRESS ANY KEY..." blocks tab-switching and ESC while awaiting. Reset-to-defaults + Back are the last two rows.

6. **Ship skins → ship types.** SHIP_PAINTS gained a `ship` field (saucer/xwing/tie/falcon/wedge/rocket/shuttle). New `drawShipBody(pc,pa,pt,type)` helper dispatches rendering; the old single-shape saucer code is the fallback. New variants added: X-Fighter (xwing), Twin-Ion (tie), Smuggler Freighter (falcon), Imperial Wedge (wedge/star-destroyer), Retro Rocket (rocket), Orbital Shuttle (shuttle). Ship-skins menu preview uses the same helper so each type is actually shown. `shipPaint.ship||'saucer'` fallback ensures old saves still work.

**Why:** User requested all of these over the course of the 2026-04-16 session as rapid-fire additions.
**How to apply:** If the user asks about "the ship I selected" or "the sun planet" or "farmland" — these are real features now. For adding more ship types, extend the `drawShipBody` switch and add a SHIP_PAINTS entry with `ship:'newtype'`.

## Session 2026-04-18 additions

8. **Solar system renamed to real planets (save-compatible).** `planetDefs` kept their stable `id` strings but got real names/colors/descriptions: mars→Mars, glimora→Jupiter (gas giant orange), ice→Uranus, lava→Mercury, sand→Venus, tomb→Saturn (`hasRings:true`, rings drawn as concentric ellipses in drawSpace), asteroid→Neptune, sun→Sun. Positions in `initSpacePlanets` reordered so the layout roughly follows solar order from Earth. Translations for sand/tomb/asteroid/sun/wormhole added to `translations.js`. **Tight-layout fix (2026-04-18 later):** first pass spread planets across tens of thousands of units — unreachable in practice. Recompressed to a compact diagonal: Earth(1500,-1500) → Mars → Jupiter → Saturn → Uranus → Neptune(14000,-6500); Venus/Mercury/Sun in -x inner direction; Wormhole(17500,-7800) as the outermost reachable body. Ship bounds tightened accordingly: x ∈ [-10000, spaceWidth+12000], y ∈ [-spaceHeight-4000, 500].

9. **Wormhole as a real planet, farthest away.** New `planetDefs` entry with `isWormhole:true` at space position `(-55000,-15000)` — the furthest body from Earth. Rendered inline in drawSpace's planets.forEach with swirl animation (uses `p._whAng`). Auto-land skipped for wormhole (`if(p.isWormhole) return;`). Old `window.wormhole` / `window.wormholeExit` / `window.voidPlanetAdded` system removed entirely.

10. **Time-travel through the wormhole → Cretaceous Earth.** Flying the ship within `wh.radius*0.5` toggles `window.prehistoricEra`, ejects the ship opposite the pull, and invalidates Earth's `savedState` so it regenerates for the new era. Message: "68,000,000 YEARS AGO — THE CRETACEOUS..." / "Back to the present day." Going through again reverts.

11. **Dinosaur-era Earth.** New `prehistoricHumanTypes` array (trex, raptor, stego, tricera, bronto, ptero) — each with `isDino:true` and a `dinoKind`. `generateInhabitant` branches on `window.prehistoricEra` before biome logic. `renderDino(h)` is called from the top of `renderHuman` when `h.isDino` — totally different silhouettes (big tailed biped for trex/raptor, 4-legged with plates/frills/long neck for stego/tricera/bronto, winged for ptero). During `prehistoricEra`: building generation skipped, military bases skipped, modern vehicles skipped, cows skipped. Zoo renders dinos correctly because the zoo tmp object spreads `...c`.

**How to apply:** Time-travel is a toggle — if the user reports "Earth is empty/weird", check `window.prehistoricEra`. To add more dino variants: extend `prehistoricHumanTypes` and add a `kind==='X'` branch in `renderDino`. To edit time-travel messaging, search for `68,000,000 YEARS AGO`.

12. **Orbits + Moon.** All solar planets now orbit the Sun. Orbit initialized in `initSpacePlanets` from each planet's starting (spaceX,spaceY) — giving orbitRadius + orbitAngle. Orbit periods use real ratios (Mercury 0.241y → Neptune 164.8y). Base period: **1 Earth year = 30 real minutes** (user-requested "much much slower"). Orbits advanced in `updateSpace` before ship bound clamping. Sun and Wormhole are stationary. Earth has a `moonX/moonY` child that orbits at radius 90 in 27.3/365 Earth-years; rendered next to Earth in drawSpace with craters + shading, and as a small grey dot on the minimap when Earth is nearby.

13. **Minimap fixes + carnivore dinos.** Minimap clips drawings to its frame rect (no more planets leaking outside) and aligns the skip rule so unlocked-but-far planets behave like the bounds calc expects. Labels now show the full planet name (`tr('planet.'+id+'.name')`), not a 4-char slice. T-Rex and Raptor dinos hunt herbivore dinos: scan for prey within 380–550 units, sprint toward them, and on bite-range contact trigger a full gore cascade — `bleedEffect` ×4, `spawnGibs` with power 2.5–4, prey marked with `eatTimer` so chunks/blood keep spraying for ~1.5s before the corpse vanishes. Hunt cooldown 300f prevents instant chain-kills. Predators don't hunt each other.

14. **Moon is a real landable planet.** Moon added to `planetDefs` between Sun and Wormhole (planetDefs index 9) with `isMoon:true`, `orbitsEarth:true`, radius 60, grey colors, 0 inhabitants, 0 buildings. Orbit init handles `orbitsEarth` specially: radius 420, period 27.3/365 Earth-years; orbit tick runs two passes (solar planets first, Earth-orbiting bodies second so Moon follows Earth's current position). Old inline "draw Earth's moon" code in drawSpace + minimap removed — the Moon now renders via the standard planet loop. Landing works through the standard auto-land path. `loadPlanet`'s while-loop for building generation short-circuits when `buildingDensity === 0` so Moon stays truly empty. Translation added: `moon: { name: 'Moon', desc: ... }`.

15. **Planets appear as lights (brighter rendering).** Removed the `'#000'` rim stop from planet `_bodyGrad` — now fades to `color2` → `color2` so no dark halo. Minimap: locked planets no longer render as `#333`; all planets draw in their actual color with a subtle halo (`color+'55'` glow behind the solid disc). Effect: every planet reads as a glowing body, even from far away.

16. **Moon surface is dedicated — craters, flag, and one weird creature (2026-04-18).** Added a Moon branch in `drawPlanet` (else-if before generic non-Earth ground): grey regolith gradient (`#bcbcc0`→`#3a3a3e`), deterministic crater ellipses with bright upper rim + dark lower lip, scattered rocks, dust speckles. No grass strokes. A rigid US-style flag (no wind — airless moon) sits at `worldWidth*0.5` with a footprint trail leading to it. A lone "moon creature" (`p._moonCreature`) wanders slowly between x∈[250, worldWidth-250]: pale translucent blob body, antenna with glowing dot, three drooping tentacles, two big sad eyes with blink animation, occasional "..." thought bubble. State persists on the `planetDefs` entry (singleton), so it survives visits. Population bug fixed: `planet.inhabitantCount||30` → explicit `!=null` check so `inhabitantCount:0` actually yields 0 humans (previously Moon was getting 30 despite the 0).

17. **Keybind swap: lasso → C, cloak → V (2026-04-18).** User remapped default canonicals so lasso uses the more reachable C and cloak moves to V. `KEY_ACTIONS` canonicals updated; keypress handler checks `k==='c'` for `toggleLasso()` and `k==='v'` for `shipCloak` toggle. If users previously rebound via settings menu, the physical→canonical map still translates correctly.

18. **Upgrade prompt text removed (2026-04-18).** The bottom-center `hud.upgradePrompt` line (shown when `score>=10`) was deleted per user — was cluttering the HUD. Replaced with a comment block at ~line 12147.

19. **Debug World: era toggle (Now / 68M yrs ago) (2026-04-18).** Press `E` on the Debug→World planet grid to flip `window._debugPrehistoric`. The era indicator ("PRESENT DAY" blue / "68,000,000 YRS AGO — CRETACEOUS" orange) renders near the bottom. `getDebugUnitCatalog(earth)` branches on the flag: when true, returns only the Cretaceous dinosaurs (`prehistoricHumanTypes`) — no modern humans/cows/vehicles. `debugUnits` title gains a `[CRETACEOUS]` suffix. Flag is a simple global; doesn't touch `window.prehistoricEra` (which is the real gameplay era used by `loadPlanet`).

**How to apply:** To view dinos in the debug unit grid, go Debug → World, press E to toggle era, then select Earth. To add more moon features, extend the `p.id==='moon'` branch in `drawPlanet` (after `}else if(p.id==='moon'){`). Moon creature state lives on the `planetDefs` entry under `p._moonCreature` — don't move it into savedState unless you want it to reset between era toggles.

20. **Earth desert: pyramids + camels (2026-04-18).** Desert biome (`generateBuilding`) now rolls `openPyramid` (4%) or `closedPyramid` (4%) in addition to the existing mosque/adobe/market/etc. `loadPlanet` for Earth guarantees at least one enterable pyramid: if the random gen produced none, upgrade a `closedPyramid` to `openPyramid`; if there are zero pyramids at all, force-spawn an openPyramid at the desert biome midpoint (`layers:5`). Tomb entry gate at ~line 9447 extended from `p.id==='tomb'` to `p.id==='tomb' || p.id==='earth'` — same puzzle interior loads with `theme='tomb'`. Camels added to `generateCows` Earth branch with new `wack:'camel'` (color `#d4a860`, spots `#8a6a3a`, size 1.1, 3–6 per load in desert biome). Dedicated `renderCow` branch draws long stilty legs with hoof dust, twin humps with sun highlights, curved neck, sleepy half-lidded eye, and tufted tail.

21. **Saturn reworked into real gas giant (2026-04-18).** Planet still has `id:'tomb'` for save compatibility but the old Khet/Egyptian-tomb theme is gone. New `planetDef` flags: `isGasGiant:true`, `hasClouds:true`, `buildingDensity:0`, `hasRings:true` (kept). Sky + ground palette shifted to banded gold/cream (`#b48840`/`#d4a868`/`#ecd098` sky, `#f0dcb0`/`#c8a870`/`#8a6830` ground). Five new `alienTypes` with `float:true` on most: Gas Whale (massive drifter), Sky Jelly, Storm Wisp, Cloud Rider (heavier, doesn't float), Aurora Sprite. New `alienLabel:'Cloud Drifter'`, cloud-themed sadFacts/cryPhrases. `generateBuilding` tomb branch emptied (dead code with buildingDensity 0); `loadPlanet` tomb pyramid/slave spawn block removed. **Saturn render branch** added in `drawPlanet` before the moon branch: `p.id==='tomb' || p.isGasGiant` → horizontal banded cloud gradient as "ground", ~9 horizontal cloud bands with slow parallax drift, atmospheric wisps near horizon, rare lightning flicker in deeper bands. **Float support** in `renderHuman`: when `h.float && !ragdoll/beam/grab`, wraps body render in `ctx.translate(0, -lift)` with a gentle 40±8 px vertical bob via `h.floatPhase`. `float` property is stored on the inhabitant in `generateInhabitant` from `template.float`.
